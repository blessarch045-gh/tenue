## 画像生成ワークフロー（現行）

**現状: Gemini Web で生成 → ユーザーが手動DL → AI がリネーム保存**

```
1. AI: 投稿予定ファイルを読んでプロンプトを準備・Geminiへ送信指示
2. USER: Gemini Web（https://gemini.google.com）で画像生成・ダウンロード
3. USER: 「ダウンロードしました」と報告
4. AI: ls -lt ~/Downloads/Gemini*.png で最新ファイルを特定
5. AI: cp でマーケティング/Instagram/画像/ にリネーム保存
```

**API自動化は準備完了・課金待ち:** `tools/generate_image.py` 完成済み。
[ai.dev/projects](https://ai.dev/projects) で請求先をリンクすれば即使用可（画像1枚 $0.02〜$0.04）。

## AI行動ルール

### 自動でやること
- ファイルの読み書き・検索・コード編集
- `ls` / `cp` / `git` 等の安全なコマンド実行
- スキルファイルを読んで手順通りに進める

### ユーザーに確認してからやること
- git push / PR作成
- 新規ファイル・フォルダの大量作成
- 既存ファイルの削除

### 即ストップ・報告する条件
- ブラウザ操作が2手順以上詰まったとき
- 金銭・課金・支払いが発生する画面が出たとき
- エラーが同じ原因で2回繰り返されたとき（同じ手段で粘らない）

## 開発ルール

- MVPスコープ外の機能は作らない（画面は6画面固定）
- カラーは `src/constants/colors.ts` で一元管理
- エラーメッセージは日本語
- 画像アップロード時: quality 0.7・長辺1200pxで圧縮
