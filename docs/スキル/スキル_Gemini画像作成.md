# スキル: Gemini 投稿画像作成

> トリガー: 「（週の）画像作って」「Geminiで画像作って」「今週の画像を作って」など

---

## 概要

Instagram投稿用の画像を **Gemini API（Imagen 3）** でバッチ生成し、
**ブラウザ操作・手動ダウンロード不要** でプロジェクトフォルダに直接保存する。

```
投稿予定ファイル → プロンプト抽出 → API呼び出し → 自動保存
```

2. `.env` の `GEMINI_API_KEY` はすでに設定済み

   ```bash
   python3 tools/generate_image.py \

   ```

## 現行の実行フロー（手動ダウンロード版）

### ステップ1: 投稿予定ファイルを読み込む

```
対象ファイル例: マーケティング/Instagram/投稿予定_Week3_3月30日〜4月5日.md
```

- ファイルから「画像プロンプト」セクションを全て抽出
- 未生成の画像（`マーケティング/Instagram/画像/` にないもの）を確認
- 生成枚数・構成をユーザーに報告してから開始

例:
```
今週の未生成画像は以下です。API生成を開始します。
- 4/1（水）カルーセル: 5枚
合計: 5枚
```

### ステップ2: Gemini Web で画像を生成

- https://gemini.google.com を開く
- 投稿予定ファイルのプロンプト＋共通スタイル指定を送信
- 生成された画像を確認

### ステップ3: 手動ダウンロード → AIが保存

```
1. 画像にカーソルを当て ↓ ボタンをクリック → ~/Downloads/ に保存
2. 「ダウンロードしました」と伝える
3. AIが cp コマンドで命名規則通りにリネーム・プロジェクト保存
```

### ステップ4: ファイル命名規則

```
[Week番号]_[月日]_[テーマ略称]_[連番とテーマ].png

例:
Week3_4月1日_春の3色ルール_1表紙.png
Week3_4月1日_春の3色ルール_2白ブラウス.png
Week3_4月5日_つくっている理由.png
```

### ステップ4: 完了報告

```
✅ 画像生成完了。以下を保存しました：
- マーケティング/Instagram/画像/Week3_4月1日_春の3色ルール_1表紙.png（2.8MB）
- マーケティング/Instagram/画像/Week3_4月1日_春の3色ルール_2白ブラウス.png（2.9MB）
（以下続く）
```

## スクリプト仕様（`tools/generate_image.py`）

| 引数 | 説明 | デフォルト |
|------|------|---------|
| `--prompt` | 画像プロンプト（必須） | - |
| `--output` | 保存先パス `.png`（必須） | - |
| `--aspect` | `1:1` / `9:16` / `16:9` | `1:1` |
| `--count` | 生成枚数（同一プロンプトで複数欲しい場合） | `1` |

- **ブランドスタイルは自動付加**: プロンプトに「水彩タッチ・tenue世界観」が自動追加される
- **保存先ディレクトリがなければ自動作成**
- **GEMINI_API_KEY は `.env` から自動読み込み**

| 要素 | 仕様 |

## よくある失敗とリカバリー

| 問題 | 対処 |
|------|------|
| `GEMINI_API_KEY が設定されていません` | `.env` にキーを設定（[取得先](https://aistudio.google.com/apikey)） |
| `画像が生成されませんでした` | プロンプトから具体的な人物描写を減らして再実行 |
| テキストが画像に混入 | プロンプトに「no text, no letters, 文字なし」を追加 |
| スタイルがリアル寄り | プロンプトに「水彩タッチ・アニメイラスト風・ソフトなタッチ」を追加 |
| カラーがブランドと合わない | 「アイボリー・ベージュ・テラコッタのみ」を明示 |
| 縦横比が違う | `--aspect 9:16` を確認 |
