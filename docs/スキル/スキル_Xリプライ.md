# スキル: X リプライ実行

> トリガー: 「今日のXリプライやって」「Xリプライ候補出して」など

---

## 概要

ターゲット固定アカウントのタイムラインをチェックし、「コーデ記録キャラクター」として返せる投稿を選んでリプライ下書きを生成する。
ユーザーが選んだものをリプライボックスに入力し、最終的な「返信」ボタンはユーザーが押す（X ToS対応・品質確保のため）。

---

## 実行フロー

### ステップ1: ターゲットアカウントをチェック

以下の順番でアカウントのプロフィールページを開いてTLを確認する。
詳細は `docs/マーケティング/X_ターゲットアカウントリスト.md` 参照。

> ⚠️ **対象は必ず公式アカウントのみ**（個人アカウントは対象外）。ブランド・アパレル・ファッションメディアの公式アカウントに限定する。

**最優先ブランド（毎日確認）:**
| アカウント | URL | 状態 |
|-----------|-----|------|
| @UNIQLO_JP | https://x.com/UNIQLO_JP | ✅ 確認済み |
| @AdametRope | https://x.com/AdametRope | ✅ 確認済み |
| @FashionsnapBuy | https://x.com/FashionsnapBuy | ✅ 確認済み |

**メディア（毎日確認）:**
| アカウント | URL | 状態 |
|-----------|-----|------|
| @fashionsnap | https://x.com/fashionsnap | ✅ 確認済み |
| @fashionpressnet | https://x.com/fashionpressnet | ✅ 確認済み |
| @SPUR_magazine | https://x.com/SPUR_magazine | ✅ 確認済み |

> ⚠️ **削除済みアカウント（存在しない・個人アカウント）**: @SNIDEL_jp、@Lily_Brown_jp、@vogue_japan、@niko_and_、@ELLE_JP は2026/3時点で存在しないか個人アカウントのため除外。
> ⚠️ GU Japan公式アカウントのハンドルは未確認。@GU_STYLEはGU日本公式ではない（ベトナム系）。

### ステップ2: リプライ候補の投稿を選ぶ

各アカウントのTLを見て、以下の基準で候補を選ぶ。

**選ぶべき投稿（3〜5件ピックアップ）:**
- 投稿時間が **24時間以内**（できれば6時間以内）
- ファッション・コーデ・スタイリング関連の内容
- いいねが **100件以上** ついている投稿
- 「着回し」「コーデ」「色」「季節」「ワードローブ」など自然にコーデ記録の話が繋がるもの

**スキップする投稿:**
- CM・広告色が強いもの（完全に宣伝だけの投稿）
- 食べ物・スイーツ・インテリアなどファッション無関係
- アニメ・コラボ（ファッション文脈でない）
- 古い投稿（48時間以上前）
- **男性向けコンテンツ**（メンズファッション・スーツ等）→ tenuеのターゲットは女性
- **子供向けコンテンツ**（キッズファッション等）→ 対象外

### ステップ3: 候補一覧を報告

スキャンした結果を以下の形式でユーザーに提示:

```
今日の候補です：

1. @UNIQLO_JP（1時間前）
   投稿内容: エアリズムの新作CM
   いいね: 1,205 / インプレッション: 24.3万
   ↓ リプライ案:
   「春になると薄着のコーデが増えて、インナー選びが一気に重要になりますよね。
   毎日コーデを記録してると、エアリズムを着始める日が毎年だいたい同じ時期で
   「あ、また春が来たな」ってなります笑」

2. @fashionpressnet（2時間前）
   投稿内容: マリメッコ春色ウニッコ柄
   いいね: 241 / インプレッション: 5.8万
   ↓ リプライ案:
   「春になると明るい色が自然と手に伸びる感覚、わかります。
   毎日コーデを記録してると、毎年3月になると選ぶ色がはっきり変わってるのが面白くて。
   自分の「春の色」って、ちゃんとあるんですよね。」

3. （以下続く）

→ どれを投稿しますか？（例：「1と3をやって」）
```

### ステップ4: リプライボックスに入力

ユーザーが番号を指定したら、その投稿に移動してリプライボックスにテキストを入力する。

```
操作手順:
1. 投稿のURLまたはアカウントから対象投稿を開く
2. 投稿ページに遷移（コメントアイコンをクリック）
3. リプライボックスをクリックしてフォーカス
4. テキストを入力
5. 「返信」ボタンが押せる状態で停止 → ユーザーに「ボタンを押してください」と伝える
```

**1件ずつ処理**（まとめて複数処理しない）。
ユーザーが「返信」ボタンを押したことを確認してから次の投稿に移る。

## リプライ作成ルール

### アイデンティティ（一貫して滲ませるキャラクター）

全てのリプライに以下の要素のどれかを自然に入れる:
- 「毎日コーデを記録してると〜」
- 「記録を見返すと〜」
- 「去年の今頃のコーデを見たら〜」
- 「何回着たか記録でわかって〜」

### 3パターンの使い分け

| パターン | いつ使う | 特徴 |
|---------|---------|------|
| **A: 体験データ型** | バズってる投稿・インプレッション高め | 具体的な数字・記録データを添える |
| **B: 視点追加型** | 共感系・Tips系投稿 | 賛同＋一段深い観察 |
| **C: 問いかけ型** | 会話を生みたい時・伸びしろある投稿 | 最後に相手への問いかけを入れる |

**詳細テンプレートは `docs/マーケティング/X_リプライ戦略.md` 参照**

### 文字数・形式

- **150〜220文字**を目安（長すぎず・短すぎず）
- 改行で読みやすくする（2〜3行構成）
- ハッシュタグは**入れない**（スパム判定リスク）
- 「tenue」「アプリ」など**宣伝ワードは禁止**

---

## NG事項（絶対やらない）

- 「アプリ作ってます」「tenue見てください」などの宣伝
- 同じ文面を複数投稿に使い回す
- ハッシュタグをリプライに入れる
- 「返信」ボタンを自分でクリックする（ToSリスク）
- 1日10件を超えるリプライ（スパム判定リスク）

---

## 1日の目安

| 項目 | 目標 |
|------|------|
| チェックアカウント数 | 最優先6アカウント（毎日） |
| 候補提示数 | 3〜5件 |
| 実際に投稿するリプライ | 3〜5本/日 |
| 作業時間 | 5〜10分（ユーザー側） |
