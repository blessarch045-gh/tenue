# 技術設計
> このファイルの用途: コード作業時にDB定義・画面構成・フォルダルールを参照する辞書。

## 技術スタック
- React Native + Expo (TypeScript) — SDK 54
- Supabase（認証・DB・Storage）
- Supabase Auth（メール＋パスワード）
- Supabase Storage バケット: tenue-photos

## 画面設計（6画面）
| 画面名 | ファイル名 | 内容 |
|--------|-----------|------|
| スプラッシュ | SplashScreen.tsx | ロゴ表示・2秒後にログインへ |
| ログイン | LoginScreen.tsx | メール＋パスワード・サインアップリンク |
| サインアップ | SignUpScreen.tsx | メール・パスワード・確認 |
| ホーム | HomeScreen.tsx | コーデ一覧グリッド・追加ボタン・ログアウト |
| 記録追加 | AddCoordScreen.tsx | 写真選択・コメント入力・保存 |
| 詳細 | CoordDetailScreen.tsx | 写真大表示・日付・コメント・削除 |

## データ設計

### profiles テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | uuid | Supabase AuthのユーザーID（主キー） |
| username | text | 表示名 |
| created_at | timestamp | 作成日時（自動） |

### coord_records テーブル
| カラム名 | 型 | 説明 |
|---------|-----|------|
| id | uuid | 主キー（自動生成） |
| user_id | uuid | profilesのidを参照（外部キー） |
| photo_url | text | Supabase StorageのURL |
| comment | text | コーデのコメント（任意） |
| worn_at | date | 着用日（デフォルト：今日） |
| created_at | timestamp | 作成日時（自動） |

### Supabase設定済み項目
- RLS有効（自分のデータのみ操作可能）
- サインアップ時のprofiles自動作成トリガー設定済み
- Storageバケット tenue-photos（公開アクセス）
- メール確認オフ（開発用）

## フォルダ構成
```
src/
├── screens/        ← 6画面のTSXファイル
├── components/     ← 共通UIコンポーネント
├── navigation/     ← AppNavigator.tsx
├── lib/            ← supabase.ts
├── hooks/          ← カスタムフック
├── constants/      ← colors.ts
└── types/          ← index.ts（型定義）
```

## MVPスコープ
含める: 認証・コーデ記録・コメント・一覧表示・詳細表示
含めない: AIコーデ提案・タグ・お気に入り・検索・ソーシャル・カレンダー・統計・プロフィール編集

## よく使うコマンド
- npx expo start — 開発サーバー起動
- npx expo start --clear — キャッシュクリアして起動
- claude — Claude Code起動
- /exit — Claude Code終了
- Ctrl + C — プロセス停止
