# X リプライ自動化 — 実現可能範囲と実装方針

> 作成日: 2026/3/16

---

## 結論：「完全自動化」はリスク大、「半自動化」が現実的

| 自動化レベル | 内容 | 実現性 |
|---|---|---|
| ❌ 完全自動（発見→自動投稿） | ボットがリプライを自動送信 | X ToS違反・アカウント凍結リスク |
| ✅ 半自動（発見→下書き生成→人間が投稿） | Claudeが候補を用意、ワンクリックで投稿 | 実現可能・安全 |

---

## なぜ完全自動化がNGか

### X利用規約の問題
- Xは「自動化されたリプライ」を明示的に禁止
- 特にアカウント開設直後（信頼スコアが低い段階）は凍結リスクが高い
- AI生成コンテンツの大量自動投稿は2024年以降厳しく監視されている

### 品質の問題
- テンプレートのみで生成したリプライは「ちぐはぐ感」が出やすい
- フォロワーを獲得するには「この人面白い」という人間的な反応が必要
- 完全自動化するとキャラクターが崩れる

## 半自動化ワークフロー（推奨）

### 所要時間：毎日5〜10分（従来の20〜30分→激減）

```
① Claudeがバズり投稿を探す（自動）
    ↓ Xを開いてファッション系投稿をスキャン
    ↓ いいね数・リプライ数を確認
    ↓ 候補5件をピックアップ

② Claudeがリプライ下書きを生成（自動）
    ↓ 投稿内容を読み込み
    ↓ A/B/Cパターンのどれが適切か判断
    ↓ 5件分の下書きをリストアップ

③ 人間がレビュー・投稿（5〜10分）
    ↓ 下書きを確認（必要なら微修正）
    ↓ XのリプライボックスにClaude側で貼り付け
    ↓ 人間が「返信」ボタンを押す
```

### 実際の運用イメージ
```
ユーザー：「今日のXリプライをやってください」
    ↓
Claude：Xを開き、候補投稿5件と下書きリプライをリストアップ
    ↓
ユーザー：「1番、3番、5番を投稿して」
    ↓
Claude：XのリプライUIを開き、テキストを入力した状態で停止
    ↓
ユーザー：「返信」ボタンを押す（1投稿5秒）
```

## 技術的に可能なこと（Claude + Chromeブラウザ操作）

| タスク | 自動化可否 | 方法 |
|---|---|---|
| バズり投稿の発見 | ✅ 可能 | Xを開いて検索・スキャン |
| 投稿内容の読み取り | ✅ 可能 | ページ内容の解析 |
| リプライ文案の生成 | ✅ 可能 | テンプレート×文脈で生成 |
| リプライボックスへの入力 | ✅ 可能 | ブラウザ操作 |
| 「返信」ボタンのクリック | ⚠️ 人間推奨 | ToSリスク回避・品質確保 |
| 投稿スケジュール管理 | ✅ 可能 | 下書きリスト管理 |

## 今すぐ始められる半自動化フロー

### 毎日の依頼コマンド（ユーザーがClaudeに言うだけ）

```
「今日のXリプライ候補を出して」
→ Claude: バズり投稿5件 + リプライ下書き5件を用意

「1番と3番を投稿して」
→ Claude: ブラウザでリプライ入力 → 人間がボタンを押す

「今週の自分発信ツイートも作って」
→ Claude: 3本分の文案を用意 → 人間が投稿
```

## 将来的な完全自動化の選択肢（βリリース後・慎重に検討）

アカウントに信頼スコアが積まれ、フォロワーが増えた段階で検討可能：

| ツール | 内容 | 費用 |
|---|---|---|
| X API Basic | 投稿・検索API利用 | $100/月 |
| Make (Integromat) | ワークフロー自動化 | $9〜/月 |
| Buffer / Hypefury | スケジュール投稿 | $15〜/月 |

⚠️ ただしリプライの完全自動化は引き続きToSリスクあり。
スケジュール投稿（自分発信ツイート）の自動化に留めるのが安全。

## 推奨：当面は半自動化で運用

- 毎日「今日のXリプライ候補出して」とClaudeに依頼
- 1日5〜10分で5件のリプライが完了
- 品質を保ちながらキャラクターを確立
- アカウントの信頼スコアが上がってから自動化を検討
